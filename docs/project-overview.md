# 專案範圍、架構與實作順序

## 問題與目標使用者

核心使用者是長時間坐著讀書、容易在專注時失去姿勢覺察的中學生。真正要解決的不是「使用者不知道要坐正」，而是：

- 專注時無法察覺姿勢已偏離。
- 固定鬧鐘與當下姿勢無關，容易被關閉。
- 沒有可解釋指標與長期回饋，使用者看不到改善。

產品承諾是「當偏移真的持續時才提醒，讓使用者看得到自己逐步改善」，不宣稱矯正脊椎、診斷疾病或取代醫療專業。

`app/` 與 `backend/` 分別是 Expo 與 FastAPI 的固定 component root；manifest 直接位於 component 根目錄。此架構不增加 project-name、framework-name 或其他分類包層。

## 系統邊界

| 邊界 | 責任 | 不負責 |
|---|---|---|
| `app` | 登入／註冊、相機權限、自動／固定視角引導、半身／全身品質、校準、即時結果、提醒、工作階段與週報／趨勢畫面 | 保存秘密、直接連 PostgreSQL、在前端呼叫具秘密的 AI API |
| `backend` | 密碼雜湊、可撤銷 session、資料 ownership、暫存影格／節點、MediaPipe Full 推論、視角／入鏡範圍、aspect-corrected 幾何、指標與規則、週報聚合、資料 API、AI provider | 永久保存原始影像、辨識旁人身分、直接控制危險硬體 |
| PostgreSQL | 工作階段、衍生姿勢 samples、提醒標記、週報所需時間欄位與分類回饋 | 原始影格、本機偏好、未經同意的聯絡人資料 |
| 量界智算 | 根據至少 10 分鐘的去識別摘要產生建議 | 接收原始影像、決定姿勢是否良好、做醫療診斷 |
| Coolify／VPS | 部署 client、API 與 PostgreSQL | 自動取得競賽 Azure 替代計分資格 |

## 決賽 MVP 垂直切片

1. 啟動時只顯示 Email／密碼登入或註冊；完成驗證後才可進入任何產品畫面。
2. 手機或 Web 可選房間自適應 `auto`、固定側面或固定正面；房間模式只處理支援區內的一位主要使用者。
3. API 以固定版本的 MediaPipe Pose Landmarker Full 取得節點，辨識 `upper_body`／`full_body`；人物過小、多人、必要節點截斷或自動視角不穩定時拒絕影格並顯示原因。
4. 引導使用者在同一已解析視角與入鏡範圍維持自然中性姿勢 10 秒，建立個人基線。
5. 每秒約 2 幀取得姿態節點，以影像寬高校正 2D 幾何；低品質影格不參與判定。
6. 半身依可見節點分析可支援的頭頸、肩線與軀幹指標；全身依視角增加側面膝屈曲或正面髖線／膝線。偏移持續 8 秒後顯示項目與調整提示。
7. 回正後停止提醒，完成工作階段並顯示良好坐姿比例、提醒次數與反應時間。
8. 正式觀察以 API 與 Alembic 管理的 PostgreSQL 保存到自己的帳號；後端聚合狀態分布、四時段與提醒次數週報，合格摘要可交由量界智算產生個人化建議。
9. 登入後可選擇展示模式；它只產生本地摘要，不建立、上傳或完成後端工作階段，也不污染歷史與週報。
10. 資料不足或量界智算不可用時顯示「規則式建議」，仍可完成主要流程且不冒充雲端結果。

## 可驗收條件

| 能力 | 決賽前完成條件 |
|---|---|
| 校準 | 自動／固定視角與半身／全身都只納入穩定同一模式的有效影格；模式或相機條件改變後重新校準 |
| 房間模式 | 單人支援區內可分析；人物過小、多人、截斷、視角不明或離開區域時明確拒絕，不宣稱任意距離保證 |
| 幾何 | 非正方形影像先做寬高校正；只有目前視角與入鏡範圍支援的指標進入分數 |
| 姿勢事件 | 只在有效影格達門檻且偏移持續 8 秒時建立事件 |
| 可解釋性 | 每次提醒至少顯示偏移項目、目前值、個人基線與門檻 |
| 介入階段 | 預設初期，可手動修改；具足夠工作階段後才允許自動升降級 |
| 資料 | 資料庫不含原始圖片；能查詢單次工作階段摘要與由衍生 samples 重現的週報狀態分布、四時段及提醒次數 |
| 帳號 | Email／密碼可註冊與登入；密碼只保存 Argon2 hash，未登入或他人 session ID 無法讀寫工作階段 |
| AI | prompt 只含摘要；量界智算、舊版 provider 與 fallback 都有明確來源標籤 |
| 展示 | 真人鏡頭與內建展示模式各能完成一次，且展示資料不進入正式歷史與趨勢 |
| 說明 | 能清楚說明支援姿勢、未支援情境、測試樣本與目前實測結果 |

## 明確不納入決賽 MVP

- 腰帶按壓、馬達施力或其他可能造成不適的硬體。
- 醫療診斷、疼痛治療、脊椎曲率量測或「預防疾病」宣稱。
- 單眼影像的公分量測、頸椎壓力倍數，以及「房間任何位置／距離都能判定」的保證。
- 人臉辨識、多人監控或跨房間追蹤；多人入鏡時本系統拒絕分析。
- 班級／企業監控、排行榜、付費訂閱與 App Store 正式上架。
- 儲存原始照片／影片、無同意的人臉辨識或將所有顧客資料直接送給生成式 AI。
- 在缺乏測試資料時宣稱準確率大於 90%、改善學習效率或降低就醫率。

## 仍需人工決定

- 主辦方在官方 Azure 關閉後接受的 25% 替代計分證據。
- 量界智算的正式 base URL、API mode、model ID、額度、資料區域、保存與訓練政策。
- Email 驗證、忘記密碼、未成年帳號同意與聯絡人通知是否納入後續範圍。
- 決賽手機型號、相機架設位置、場地網路與展示時間。
- 房間單人支援區的實際距離、人物最小尺寸、auto 視角與全身新指標門檻，須由真人與決賽設備測試後凍結。
