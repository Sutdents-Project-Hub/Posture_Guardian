# 第六屆中學生黑客松競賽策略

## 已確認規則

- 主題：生活化人工智慧加值雲端應用。
- 作品需解決食、衣、住、行、育、樂等生活議題，並包含主辦方指定雲端資源與 AI 元素。
- 主賽事決賽：2026-07-26，高雄女中。
- 現場需進行作品簡報、展示與問題詢答。
- 作品須為學生積極參與創作，不得抄襲或把創作全交給 AI。

來源：[第六屆中學生黑客松官方網站](https://sites.google.com/view/hsh2026/)。主辦方可能更新規則，決賽前需再次核對。

## 決賽評分對應

依訓練營提供的決賽評分截圖：

| 評分項目 | 比重 | 本專案必須提供的證據 |
|---|---:|---|
| 簡報表達能力 | 35% | 一條流暢故事、真人 demo、備援影片、清楚回答限制 |
| 概念驗證（Azure） | 25% | 真實 Microsoft Foundry 資源／呼叫、輸入摘要、回應與 log；不可只放 Azure logo |
| 企劃完整度 | 20% | 姿勢標準、資料流程、隱私、測試方法、失敗備案與開發範圍 |
| 問題理解度 | 10% | 聚焦中學生專注時缺乏覺察，不使用未查證盛行率 |
| 創新性 | 10% | 個人基線 + 可解釋指標 + 改善程度介入 + AI 摘要建議 |

## 建議敘事

1. **問題**：小明讀書很專心，卻總在肩頸不舒服後才發現姿勢已偏移。
2. **現有缺口**：固定鬧鐘不看姿勢，護具只提供物理限制，也無法說明改善。
3. **洞察**：真正需要的是「偏移持續時才提醒」與「看得見自己改善」。
4. **方案**：手機／Web 相機 + MediaPipe 姿態節點 + 個人校準規則 + PostgreSQL 趨勢 + Microsoft Foundry 建議。
5. **證據**：展示同一個人的基線、偏移角度、8 秒事件、回正與工作階段摘要。
6. **誠實限制**：這是姿勢覺察工具，只支援指定視角與姿勢，不量測真實脊椎曲率。

## Demo flow

實際秒數待主辦方公告後裁切，但順序固定：

1. 顯示首頁與隱私承諾：原始影像不保存。
2. 選側面模式，完成 10 秒校準。
3. 畫面顯示耳、肩、髖節點與目前／基線角度。
4. 示範前傾 8 秒；觸發「頭頸偏移」提醒，而不是模糊的「你坐姿很差」。
5. 使用者回正；結束工作階段，顯示良好坐姿率、提醒次數與平均分數。
6. API 儲存摘要，Foundry 依主要偏移與近期比例產生一段個人化建議。
7. 切換至技術證據畫面：PostgreSQL 記錄、Azure 呼叫／log、fallback 標籤與架構。

## 使用者情緒體驗地圖

| 階段 | 情緒 | 系統行為 | 成功訊號 |
|---|---|---|---|
| 專注讀書 | 無感、投入 | 安靜觀察，不頻繁打斷 | 使用者忘記系統存在 |
| 偏移持續 | 微疲勞、未察覺 | 8 秒後溫和說明偏移 | 提醒與當下狀況一致 |
| 收到提醒 | 可能被打擾 | 顯示原因、可稍後提醒 | 使用者不覺得被責備 |
| 回正 | 放心、有掌控感 | 立即停止提醒並給正向回饋 | 快速回到學習 |
| 查看趨勢 | 好奇、被鼓勵 | 顯示真實改善與一項可行建議 | 願意開始下一次工作階段 |

## 不可沿用的舊簡報主張

- 「Azure AI Vision 可辨識人體骨架」：官方 Image Analysis 只列人員框、物件、標籤等能力，且 4.0 已公告淘汰時程；姿態節點改用 MediaPipe。
- 「83% 青少年姿勢不良」、「70% App 一個月放棄」、「效率下降 20-30%」：來源未能驗證，刪除。
- 「肩膀高低差超過 2cm」：單眼 2D 影像沒有尺度校準，改用角度與個人基線。
- 「準確率 > 90%」與「降低就醫 20%」：未實測前只能列目標，不能列成果。
- 「唯一整合方案」：未做完整專利／市場調查，不使用絕對化宣稱。
- 腰帶按壓：硬體不是實驗主旨，MVP 改為手機震動／聲音；不做可能造成不適的施力裝置。

## 現場失敗備案

| 失敗 | 備案 | 揭露方式 |
|---|---|---|
| 相機權限或光線失敗 | 首頁啟動內建「展示模式」的合成骨架流程 | 全程標示「展示模式」 |
| 場地網路失敗 | 展示模式在 client 執行校準、事件與離線摘要 | 畫面標示「離線摘要」，不宣稱當下雲端回應 |
| Microsoft Foundry 不可用 | 規則式 fallback 建議 | 畫面標示 provider=`fallback` |
| PostgreSQL 不可用 | client 暫存當次摘要，停止歷史功能 | 明確顯示未同步 |
| 真人姿勢未觸發 | 顯示節點品質與門檻，再改用測試片段 | 不偷偷降低門檻 |
