# 第六屆中學生黑客松競賽策略

## 已確認規則

- 主題：生活化人工智慧加值雲端應用。
- 作品需解決生活議題，並包含 AI 與雲端元素。
- 主賽事決賽：2026-07-26，高雄女中。
- 現場進行作品簡報、展示與問題詢答；作品必須有學生積極參與。

來源：[第六屆中學生黑客松官方網站](https://sites.google.com/view/hsh2026/)與[官方決賽評分 PDF](https://drive.google.com/file/d/1MZ11mB2PYYHQyLc-nqDYFQM_DivqnOcP/view?usp=sharing)。決賽前必須再次核對最新公告。

## Azure 關閉後的架構決策

使用者於 2026-07-16 告知「官方 Azure 已關閉」，因此實作改為：自有 VPS + Coolify + PostgreSQL + 量界智算。這是目前的部署事實來源，不等同於已取得主辦方「可替代 Azure 計分」的公開公告。

若簡報需要說明 Azure，只能使用「Azure 概念對應（非實際使用服務）」：MediaPipe Full／FastAPI 對應自訂電腦視覺推論工作負載，Coolify containers 對應應用託管角色，PostgreSQL 對應 Azure Database for PostgreSQL 類型角色，量界智算對應生成式 AI 建議角色，自建 Argon2／bearer auth 對應身分存取角色。這些都不是 Azure Resource 或成功呼叫證據；完整表述以 [整合文件](integrations.md#azure-概念對應非實際使用服務) 為準。

決賽現行展示請使用 [VPS／Coolify 現行架構版簡報](../outputs/Posture_Guardian_概念驗證_現行架構版.pptx)。`outputs/Posture_Guardian_概念驗證.pptx` 僅保留作歷史 Azure POC，不可作為目前架構或已完成部署的證據。

官方決賽評分 PDF 仍把「概念驗證（Azure）」列為 25%。量界智算與自有 VPS 能證明完整雲端產品與 AI 整合，但不能自行宣稱等同取得 Azure 25%。決賽前最高優先事項是保存官方關閉通知，並向主辦方取得下列書面確認：

1. Azure 關閉後，25% 項目的替代評分標準。
2. 自有 VPS／Coolify／PostgreSQL／第三方 AI 是否可作等價證據。
3. 需提交哪些截圖、log、架構圖或現場呼叫。

## 目前獲獎準備度

- 程式垂直切片、離線展示、隱私邊界、嚴格輸入、AI fallback、Dockerfile／本機 Compose 整合與 migration 已具可重現證據；正式部署規格為三個獨立 Coolify Resources。
- MediaPipe Full、自動／固定視角、半身／全身、單人房間品質拒絕、aspect-corrected 幾何與週報已進入程式整合；自動化測試、本機建置、真人支援區與決賽設備效能必須分開列證據，未完成者不可寫成已驗證。
- 真實 Coolify VPS、DNS、HTTPS、PostgreSQL backup／restore、量界智算 endpoint／model／成功呼叫仍待外部驗證。
- 最大評分風險是 Azure 25% 的替代資格，而不是前端功能；未取得主辦方確認前必須在答辯中誠實說明。
- 簡報 35% 仍需計時簡報、真人 demo、備援錄影、全員 Q&A 與完整彩排。
- 3–5 位知情同意真人測試尚未完成，不可宣稱準確率或改善成效已達標。

## 建議敘事

1. **問題**：中學生讀書投入時，常在感到不適後才注意到姿勢已偏移。
2. **現有缺口**：固定鬧鐘不看當下姿勢，也無法解釋長期變化。
3. **洞察**：真正需要的是「偏移持續才提醒」與「看得見自己的改善」。
4. **方案**：手機／Web 相機 + MediaPipe Full 節點 + 自動／固定視角與半身／全身個人校準規則 + PostgreSQL 週報趨勢 + 量界智算建議。
5. **工程應變**：官方 Azure 關閉後，團隊把部署轉到自有 VPS，仍保留可稽核、安全降級的完整雲端架構。
6. **限制**：工具只做姿勢覺察，支援指定視角，不量測真實脊椎曲率，也不作醫療診斷。

## Demo flow

1. 首頁說明原始影像不保存；正式觀察先登入，或選擇不需帳號的展示模式。
2. 選擇「房間自適應」，說明它只支援固定鏡頭單人區域；或切換固定側面／正面模式。
3. 讓一位使用者進入支援區，顯示解析視角、`upper_body/full_body`、距離與品質；以預先排練方式展示過遠／多人拒絕，不拍未同意旁人。
4. 完成 10 秒個人校準，顯示節點、aspect-corrected 2D 角度、個人基線與門檻。
5. 半身展示頭頸／軀幹；全身側面展示膝屈曲，或全身正面展示髖線／膝線。維持偏移 8 秒後觸發具原因的提醒。
6. 回正後結束工作階段，顯示良好坐姿率、提醒次數與平均分數。
7. 場地網路或相機失敗時改用內建展示模式：只顯示本地摘要，不建立後端工作階段或污染正式歷史。
8. 正式觀察的 API 以 Alembic 管理的 PostgreSQL 保存去識別摘要。
9. 合格資料達 10 分鐘時，量界智算產生「趨勢、下一步、下次目標」；失敗則透明顯示 fallback。
10. 週報展示可重現的狀態分布、凌晨／上午／下午／晚上四時段與 reminder count；建議只重用已持久化的安全 session insight，或顯示 deterministic fallback，重新整理不重複呼叫外部 AI。
11. AI 洞察展示六次趨勢、資料充分度、model ID、API mode、prompt version 與實際 provider。
12. 展示 Coolify 三個 services、健康狀態與去敏成功 log；不可只放供應商 logo。

## 情緒體驗地圖

| 階段 | 情緒 | 系統行為 | 成功訊號 |
|---|---|---|---|
| 專注讀書 | 無感、投入 | 安靜觀察 | 使用者不被頻繁打斷 |
| 偏移持續 | 未察覺 | 8 秒後溫和說明偏移 | 提醒符合當下狀況 |

## 不可宣稱

- 「量界智算等於 Azure 25%」：除非主辦方已書面確認。
- 「AI 判斷坐姿」：坐姿判定由 MediaPipe 節點與規則引擎負責，生成式 AI 只把摘要轉成建議。
- 「肩膀高低差超過 2cm」：單眼 2D 沒有尺度校準，應用角度與個人基線。
- 「偵測脊椎曲率／頸椎承受 1.5 倍或 2 倍壓力」：本系統只有 2D 姿態 proxy，沒有醫療影像、力學感測器或臨床驗證。
- 「房間任何地方都能偵測」：只能說固定鏡頭的單人支援區；人物過小、多人、截斷、遮擋或視角不明會拒絕。
- 「物理按壓矯正」：產品只提供視覺與可關閉的手機震動，不使用腰帶、氣囊或馬達施力。
- 「已使用 Azure AI Vision／App Service／Functions／Cosmos DB／Azure OpenAI／Azure AD」：只能展示概念對應，不得稱為實際服務。
- 「準確率 > 90%」、「降低就醫 20%」或未驗證盛行率：真人驗證與可信來源不足。
- 醫療診斷、治療、矯正脊椎或傷害機率。

## 現場失敗備案

| 失敗 | 備案 | 揭露方式 |
|---|---|---|
| 相機權限／光線失敗 | 內建展示模式 | 全程標示「展示模式」 |
| 場地網路失敗 | client 執行展示校準、事件與離線摘要 | 不宣稱當下有雲端回應 |
| 量界智算不可用 | 規則式 fallback | provider=`fallback` |
| PostgreSQL 不可用 | client 保留當次本地摘要，停用歷史 | 顯示未同步 |
| 真人姿勢未觸發 | 顯示品質與門檻，再用展示模式 | 不臨場偷偷降低門檻 |
